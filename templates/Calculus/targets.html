О проекте

Актуальность

  • Задачи решаемые в курсе безопасности жизнидеятельности применяются на реальном производстве.
  • Каждая задача состоит из большого количества трудоемких расчетов, в связи с чем намного проще алгоритмизировать процесс ее решения
  • Реализация проекта в виде сайта имеет ряд значительных плюсов, таких как:
    • Отсутствие необходимости устанавливать сторонее ПО на устройство
    • Доступ из любой точки мира и на любом девайсе
    • Мы не собираем никаких личных данных своих пользователей

Цели

  1. Создать удобный ресурс на котором студенты МИСиСа смогут проверять правильность выполнения практических работ по БЖД 
  2. Веб-ресурс должен быть адаптирован под разные устройства 
  3. Сделать красивое минималистическое оформлениe 
  4. Создать базу данных, для хранения запрос и результатов 
  5. Создать быстрые и легко тестируемые алгоритмы решения задач 
  6. Написать тесты 
  7. Развернуть и запустить веб-приложение на сервере 

Технологии

  • Python - Объектно-ориентированный язык программирования с множеством библиотек.
  • Html и CSS - Оформление и внешний вид страниц нашего веб-ресурса.
  • Bootstrap - это набор инструментов для создания сайтов и веб-приложений. Именно он позволил в короткие сроки написать полностью адаптивный сайт.
  • SQLite3 - Для хранения результатов расчетов
  • Django - Фреймворк для веб-приложений на языке Python, использующий шаблон проектирования MVC(Model-View-Controller)
  • Django ORM- Позволяет нам взаимодействовать с базой данных, используя код Python, а не SQL. Да и это, наверное, главная и мощная особенность фреймворка Django

Основные этапы разработки

Написание версток страниц сайта

  • Продумывание взаимодействия с пользователем
  • Первые протопипы страниц
  • Написание версток страниц сайта с использованием Bootstrap
  • Добавление CSS
  • Адаптация под мобильные устройства

Написание алгоритмов решения задач

  • Детальный анализ каждого пункта задания
  • Перенос всех таблиц в код
  • Нахождение функций интерполирования и экстраполирования
  • Написание скриптов для вычисления каждого пункта практической работы
  • Создание удобного интерфейса взаимодествия для интеграции в бекенд часть
  • Написание unit-тестов, для каждой задачи по 5-ти разным вариантам с известными заранее ответами

Написание Бекенд части

  • Продумывание архитектуры хранения данных, написание моделей, проведение миграций.
  • Создание базового шаблона и наследование остальных шаблонов от него
  • Написание контроллеров представлений и маршрутизации
  • Написание форм и интеграции в них скриптов для расчета задач
  • Представление результата запроса

Тестирование ПО

  • Тесты написаны для алгоритмов вычисления задач, так и отдельно для backend части приложения
  • Код покрыт тестами для простого расширения проекта

Заключение

По итогам работы, самым сложным оказалось разработать адекватный алгоритм расчёта данных по задачам. Ввиду дискретности таблиц, алгоритм может выдавать некрасивые значения при экстраполяции и интерполяции – программа работает нормально только в пределах определённого диапазона входных данных, соответствующим вариантам в пособии. Проект легко расширять благодаря архитектуре, полученной от django, и уже написанным unit-тестам.
